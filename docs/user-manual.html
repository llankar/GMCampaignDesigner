How to use this manual

Start with the six workflows below. Each one carries a real task from its starting point to a checkable result. Use the reference appendix when you need the controls and options for one particular feature.

Screenshot age: Screenshots are generated examples and may lag behind the current interface. When wording differs, follow the menu paths and button labels in the text.

Install and Create Your First Campaign

Outcome: the application opens with a new campaign database, a selected game system, and an initial backup.

  1. Install or unpack the application. For a packaged release, keep the complete application folder together and run RPGCampaignManager.exe. From a source checkout, install Python 3.8 or newer, run python -m pip install -r requirements.txt, then run python main_window.py from the repository root.
  2. Create the database. Choose File → Change Database or press F6. Enter a new campaign name and click Create; select the new database and click Use Selected. The campaign is stored under Campaigns/<campaign-name>/ unless you browse to an external database.
  3. Choose the rules system. Use Switch System at the top of the sidebar. Confirm the system name and theme shown in the main window.
  4. Add the first records. Open Campaign → PCs and create at least one player character, then add a Place and an NPC from the corresponding managers. Save each editor before closing it.
  5. Take a baseline backup. Choose File → Create Backup, select a safe destination outside the active campaign folder, and keep the resulting archive.

Checkpoint: close and reopen the application, select the campaign if prompted, and confirm that the PC, Place, and NPC are still present.

Build and Organize a Campaign

Outcome: campaign arcs, scenarios, and reusable entities form a navigable structure.

  1. Sketch the campaign structure. Open GM Tools → Campaign Builder. Define the campaign premise and create or edit arcs before generating detailed content.
  2. Create the shared cast and locations. Add recurring NPCs, factions, places, objects, clues, and maps from Campaign. Use stable, distinct names because scenario links resolve through these records.
  3. Add scenarios to the right arc. Create them manually from Campaign → Scenarios, use the Scenario Builder Wizard, or follow the AI workflow below. Review linked entities and the campaign/arc assignment before saving.
  4. Visualize relationships. Use a character, faction, or scenario graph for relationships; use Scene Flow for the order and branches between scenes; use World Map for geographic links.
  5. Review the overview. Open Campaign → Campaign Overview, select an arc and scenario, and correct missing or misplaced content before session prep.

Checkpoint: the Campaign Overview shows at least one arc containing a scenario with linked people and places.

Turn an Idea into a Playable Scenario

Outcome: a reviewed scenario contains a hook, scenes, stakes, and valid links to campaign entities.

  1. Choose an authoring path. Use GM Tools → Scenario Builder Wizard for guided manual planning, Generate Scenario for random or AI-assisted drafting, or Import Scenario for existing material.
  2. Define play-facing essentials. Record the hook, objective, stakes, secrets, expected scenes, and likely outcomes. Link existing NPCs and Places instead of duplicating them.
  3. Review generated or imported content. Check names, scene headings, relationships, and any new records before saving. AI output is draft material, not verified campaign truth.
  4. Arrange the scene flow. Open Scene Flow, order the nodes, label alternate routes, and ensure the group can recover from a skipped scene or failed clue.
  5. Verify references. Run Campaign → Verify Campaign and resolve missing or ambiguous links before building the session workspace.

Checkpoint: open the scenario detail view and read it from hook to outcome without needing an unlinked or undefined record.

Prepare a Session with the GM Screen and GM Table

Outcome: the GM workspace contains the scenario, rules references, handouts, maps, notes, and utilities needed at the table.

  1. Choose the session scenario. Open its detail view and check scenes, cast, locations, clues, and handouts one last time.
  2. Build the GM Screen. Open Campaign → GM Screen, add tabs for frequently referenced entities, random tables, scene flow, or other tools, and save the layout.
  3. Arrange the GM Table. Open Campaign → GM Table, select the scenario, and add panels for scenario detail, handouts, Map Tool, Image Library, notes, timers, or other session material.
  4. Separate private and player material. Keep secrets in clearly marked GM-only panels. Inspect every panel before using Reveal or placing it in a fixed overlay.
  5. Save and rehearse. Save the GM Table layout, test minimized-panel restore, open the important maps, and confirm audio and timer controls before players arrive.

Checkpoint: reopen both workspaces and confirm the layout returns without exposing GM-only notes on the player display.

Present Maps, Handouts and Ambiance

Outcome: players receive only the intended map state, image, handout, whiteboard, or ambiance.

  1. Choose the display route. Use a second-monitor player view for local play or a web display for remote devices. Treat every player display as public to the table.
  2. Prepare maps privately. Load the background, add tokens and drawings, set facing and measurements, and paint fog of war before broadcasting. Test pan and zoom synchronization.
  3. Reveal handouts deliberately. Open the handout or Image Library panel, inspect it for hidden notes and filenames, then reveal only the selected content.
  4. Set sound and visual ambiance. Build Music, Ambience, and SFX playlists in GM Tools → Sound & Music. Use Ambiance Control separately for player-facing wallpapers.
  5. Secure remote access. Use an access token and a trusted network. Do not expose the web server directly to the public internet; disable remote editing when players do not need it.

Checkpoint: view the player display from the players' position or device and verify that no GM notes, unrevealed map area, or private campaign media are visible.

Back Up, Validate and Repair a Campaign

Outcome: a restorable backup exists and campaign references pass validation after any repairs.

  1. Create a fresh backup. Choose File → Create Backup before importing, generating content in bulk, changing schemas, or applying repairs. Store the archive outside the active campaign directory.
  2. Run validation. Choose Campaign → Verify Campaign. Work through missing references, ambiguous matches, and invalid campaign/arc/scenario relationships.
  3. Apply the smallest repair. Replace a broken reference with the intended record or remove it when it is genuinely obsolete. Review the summary before committing changes.
  4. Re-run validation. Confirm the repaired campaign is clean, then open affected scenarios and entity detail windows to spot-check the result.
  5. Restore only when needed. File → Restore Backup replaces active campaign data and assets with the selected archive. Back up the current state first, even when it is damaged, so the restore can be reversed.

Checkpoint: validation reports no unresolved problems and a copy of the tested backup is stored separately from the campaign.

Reference Appendix

The remaining sections describe individual windows, controls, menus, integrations, and shortcuts. They are reference material rather than a required reading order.

Getting Started Reference

  • Launch the app: python main_window.py.
  • Open File → Change Database or press F6 to choose or create a campaign database.
  • Use Switch System (top of the sidebar) to choose the campaign rules system and a visual theme.
  • Use Help → Guided Tour for interactive onboarding.
  • Populate PCs, NPCs, Creatures, Places, Objects, Information records, Clues, Maps, and Books from Campaign or the Campaign Workshop sidebar.
  • Create a backup from File → Create Backup before a large import, restore, AI generation, or validation repair.

Navigation

The top menu is the stable route used throughout this manual; the sidebar provides faster access to many of the same commands.

  • File: database selection, backups, systems, schemas, AI Settings, shared assets, and Auto Improve App.
  • Campaign: entity managers, GM Table, GM Screen, Campaign Overview, World Map, graphs, scene flow, random tables, and Verify Campaign.
  • GM Tools: scenario generation, imports, exports, portraits, image library, map, whiteboard, dice, audio, timers, ambiance, and character creation.
  • View: compact Audio Bar and Dice Bar.
  • Help: Guided Tour and the shortcut reference.

Sidebar Accordion

The sidebar groups every command inside collapsible sections. Hover a blue header to expand its button grid; moving the pointer away collapses it after a short delay. The Campaign Workshop section reopens automatically when idle.

  • Switch System: Choose the rules system and visual theme for this campaign.
  • Manage Campaign Systems: Edit system dice defaults, supported faces, and analyzer config.
  • Customize Fields: Open the custom field editor (see Editor Tools).
  • New Entity Type: Add custom entities and templates.
  • AI Settings: Configure the AI endpoint, model, temperature, token limit, and optional API key.

  • Create/Restore Campaign Backup: Save or recover a full campaign archive.
  • Cross-campaign Asset Library: Export/import NPCs, Objects, and Maps with media bundles.
  • Set SwarmUI Path: Point the portrait generator at your SwarmUI installation.

Systems & Data

Cross-Campaign Asset Library

The Cross-campaign Asset Library is the transfer hub for moving reusable content between campaigns without manually copying database rows or media folders.

  • Browse source campaigns: Select a neighboring campaign from the left rail and inspect its NPCs, objects, maps, and other supported entities.
  • Preview before transfer: Each tab shows the selected record's name, summary, and image preview so you can confirm what will be exported or copied.
  • Export bundles: Export Selected… packages chosen records and their media into a portable zip bundle.
  • Copy directly: Copy to Current Campaign imports selected records straight into the active campaign and resolves duplicates interactively.
  • Import bundles: Import Bundle… restores a shared bundle; Import Image Library… limits the import to image-library assets.
  • Workspace extras: Full campaign bundles can carry GM Table layouts and ambiance wallpapers in addition to entity records and media.
  • Duplicates: Review merge prompts carefully. Make a backup first when importing into a populated campaign.
  • Online gallery: Publish bundles to GitHub, browse online releases, download shared bundles, or install a full campaign package. Publishing actions require a configured GitHub token.

Editor Tools

The Generic Editor window opens when you add or double-click an item. Fields are generated from the active template and include quick actions:

  • Action bar: Save, cancel, and scenario-aware AI buttons appear based on the entity type.
  • Rich text: Long-form fields use the RichTextEditor with inline toolbars and AI helpers.
  • Linked lists: Multiselect comboboxes let you associate NPCs, Places, Factions, and more.
  • Portraits & files: Manage artwork attachments directly from the editor.

Portrait workflow: Add multiple portraits, set a primary portrait, search the web for images, paste from the clipboard, or generate art via SwarmUI (requires the SwarmUI path in Data & System).

Portrait Generation

SwarmUI-backed portrait tools are available from entity editors, the GM Tools menu, and scenario context menus.

Generate portraits for a scenario

  1. Open Campaign → Manage Scenarios, right-click a scenario, and choose Generate Portraits….
  2. Select the linked NPCs, Places, Creatures, or other supported entities to process.
  3. Choose the SwarmUI model, number of candidates, and CFG scale. More candidates take longer and use more generation resources.
  4. Generate missing portraits individually or use Generate All Missing. When several candidates are returned, select the image to keep.
  5. Confirm that the selected portrait appears on the entity before closing the manager.

Bulk and manual workflows

  • GM Tools → Generate Portraits: generate missing NPC or Creature portraits in bulk.
  • GM Tools → Import Portraits from Folder: match existing image files to portrait-capable records.
  • Entity editor: add several portraits, set the primary image, paste from the clipboard, search the web, or generate a new image.
Before generation: Set the SwarmUI installation with File → Set SwarmUI Path. Generated filenames are sanitized and selected images are copied into the active campaign.

Random Tables

Create and roll random tables for inspiration or procedural prep. You can open the editor from Relations & Graphs → Create Random Table, or add a Random Tables panel inside the GM Screen.

  • Browse & filter: Filter by category, style, or tag and preview table metadata.
  • Rolls: Roll once or multiple times; results are stored in a running history.
  • Edit & import: Edit tables, import entries from text, or jump to the built-in Plot Twists table.

  • Drag to arrange: Left-click and drag nodes to reposition; mouse wheel zooms the canvas.

Campaign Builder

The Campaign Builder Wizard is a structured campaign-planning flow for foundation, arc planning, AI-assisted scenario generation, and final review.

  • Foundation step: Define campaign identity with name, genre, tone, status, dates, logline, setting, objective, stakes, themes, notes, and optional presets.
  • Arc planner: Build a library of arcs, edit each arc's summary/objective/thread, attach scenarios, and reorder or duplicate arcs.
  • AI operations: Generate arcs from scenarios, generate scenarios per arc, run DB-aware validation, preview forge output, and tune generation defaults.
  • Review & save: Inspect the compiled campaign payload before saving it back into the campaign database.

Campaign Overview

The campaign overview presents a visual, campaign-facing summary built from arcs, scenarios, linked entities, and briefings. It is useful for reviewing pacing, campaign structure, and scenario dependencies without opening every editor.

  • Campaign hero: Review the top-level campaign identity, tone, stakes, and poster-export entry point.
  • Arc navigation: Move across arcs with the stepper and strip while keeping summaries and objectives visible.
  • Scenario focus: Drill into each scenario's hook, objective, stakes, scene count, and link density from the same screen.
  • Entity browser: Open related NPCs, creatures, places, factions, and villains directly from the scenario sidebar.
  • Campaign-facing use: This view is for overview and navigation; detailed live-session prep still lives in the GM Screen and GM Table.

GM Virtual Table

The GM Table is the freeform virtual tabletop-style workspace for arranging panels, maps, handouts, and utilities around a single scenario.

  • Open: Choose Campaign → GM Table, then select a scenario. The default layout opens scenario details and handouts side by side.
  • Content: Add Campaign Dashboard, World Map, Map Tool, Scenario Board, Scene Flow, Image Library, Handouts, Container Window, Loot Generator, Object Shelf, Whiteboard, Random Tables, Plot Twists, entity details, Puzzle Display, Note Tab, Sticky Note, and graph panels.
  • Organization: Search, tile, cascade, align, distribute, resize, snap to grid, or spread panels. Sticky notes can be clustered by tag or colour; minimized panels remain available in the restore tray.
  • Annotations and attachments: Draw desk annotations and attach campaign images or entity material to the workspace.
  • Player reveal: Reveal a selected panel to the player display. Fixed overlays remain positioned above the player view until removed.
  • Appearance: Panel skins and depth styles help visually group material.
  • Persistence: Save the table layout after arranging panels. Layouts persist per scenario; named table layouts can be transferred in full campaign bundles.
Player-facing safety: Check a panel for GM-only notes and secrets before revealing it or placing it in a fixed overlay.

GM Screen

The GM Screen consolidates scenario prep: select a scenario to open tabs for NPCs, Places, scenes, synopsis, secrets, and notes. Use Ctrl+F for instant search, toggle the PC banner for quick reference, and click any linked entity to open its detail frame.

  • PC banner: A one-line, scrollable summary of the party that follows the active theme.
  • Section sidebar: The vertical Sections navigator lets you jump between summary, scenes, synopsis, secrets, notes, and linked-entity views.
  • Panels: Add panels such as Scene Flow, Random Tables, or World Map via the layout manager.
  • Scenario scenes: Scenes can be expanded/collapsed individually and show description, linked entities, maps, and scene-to-scene links.
  • Premium entity cards: NPCs, Places, Creatures, and Villains are displayed as dashboard cards with portraits, chips, and rich text sub-panels.
  • Quick navigation: Collapsible scenes, linked tables, and detail pop-outs keep prep focused.

Scenario Tools

Scenario toolkit for rapid authoring:

  • Scenario Builder Wizard: Plan scenes step-by-step, link NPCs/Places/Maps, and preview a scene flow before saving.
  • Scenario Generator: Use either the random generator or AI prompt generator, select a saved prompt and AI model, answer guided questions, then review the streamed result before saving.
  • Scenario Importer: Map headings from external documents into template fields before saving.
  • AI & web import: Use AI helpers and the Web Text Import flow to turn online sources or external text into structured scenario content.
  • PDF Importers: Utilities include Creature and Equipment importers that extract entries from PDFs with a review step.

AI Scenario Generation

  1. Configure the endpoint from File → AI Settings. For Ollama, ensure the service is running and at least one model is installed.
  2. Open GM Tools → Generate Scenario and choose AI prompt generator.
  3. Select a prompt and a discovered model. The last model is remembered. Use Manage Prompts to create, edit, duplicate, or remove reusable guided prompts.
  4. Answer the prompt questions and start generation. Output streams into the result area while the request is running.
  5. Review names, scene headings, linked entities, traits, and descriptions before saving.
  6. When saved, structured output can create the scenario and missing linked entities such as NPCs and Places. Existing entity names are normalized to reduce broken links and duplicate records.
Review required: AI output can be incomplete or incorrect. Back up the campaign before a large generation and inspect newly created entities afterward.

Campaign Validation

Open Campaign → Verify Campaign to check campaign hierarchy and entity references.

  • The validation wizard reports missing references, ambiguous matches, and invalid campaign/arc/scenario relationships.
  • For a missing reference, remove it or select a replacement target.
  • For an ambiguous reference, inspect the source field and choose the intended record.
  • Review the summary before applying fixes. Re-run verification afterward to confirm the campaign is clean.
Backup first: Validation repairs modify campaign data and may update several references.

Campaign Time & Session Tools

  • Advance Timeline: Open from the Utilities sidebar to move campaign time forward and process scheduled events.
  • Calendar: Use the Calendar control in the main window to show or hide the campaign calendar dock; open the full calendar for event editing and navigation.
  • Session Timers: Choose GM Tools → Session Timers or the main Timer control to run session countdowns and reminders.
  • Character Creation: Choose GM Tools → Character Creation to build a PC using the active system's rules and progression options.

Scene Flow

Visualize your scenario as a flow of scenes and links. Drag nodes to rearrange, connect scenes with labeled links, and preview the structure before a session.

  • Open: Campaign → Scene Flow Viewer, the Relations & Graphs sidebar, or a GM Screen/GM Table panel.
  • Nodes & links: Create, rename, colorize scenes; add directional links with labels.
  • Layout: Pan and zoom the canvas; arrange scenes for readability.

Map Tool

The Map Tool opens in its own window so you can prep encounters while the campaign lists stay visible. Use the selector view to choose or import a battle map, then switch to the editor to reveal fog, drop tokens, and broadcast to players.

  • Map selector: Browse the maps table, double-click to load, or right-click to import directories of images.
  • Fog of war: Paint additive or subtractive fog with brush shortcuts ([/]) and reset the mask with a single click.
  • Tokens & auras: Add NPC, PC, or creature tokens, colour their borders, track HP overlays, and duplicate or delete entries through the context menu.
  • Token facing: Rotate token facing where the selected token and view support it.
  • Drawing tools: Switch between Token, Rectangle, and Oval modes to sketch zones, spell areas, or light auras with filled/outline styles; add editable text labels and tweak drawing colours.
  • Background rotation: Rotate the map image from the toolbar; rotation is saved with the background for consistent sharing/exports.
  • Broadcast & sync: Mirror the current map to fullscreen or the web client; pan and zoom updates are pushed live.
  • Remote display: Treat the web display as player-facing. Enable remote editing only on a trusted network and use the configured access token.

Whiteboard

The Whiteboard gives you a synchronized canvas for prep and live collaboration. Use it in the GM view, pop a player view to a second screen, or share a web link/QR code.

  • Tools: Pen, eraser, stamps, and editable text (double-click to edit); pick colours, sizes, and layers.
  • Layers & grid: Toggle shared/GM layers, enable grid overlay, and snap strokes or stamps to the grid.
  • Pan & zoom: Middle-drag to pan, mouse wheel to zoom; the canvas extends infinitely.
  • Save/load: Whiteboard states are stored in the database with autosave; load previous boards instantly.
  • Player view: Open a fullscreen view on another monitor or share via QR/link for remote collaboration.

World Map

The World Map window lets you navigate nested maps, place NPC/PC/Creature/Place tokens, and drill down to regional maps while reviewing a compact inspector for the selected entity.

  • Open: Campaign → World Map, the Relations & Graphs sidebar, or a GM Screen/GM Table panel.
  • Select map: Load an existing entry or create a new one and assign a background image.
  • Tokens: Add NPCs, PCs, Creatures, Places, and Maps as pins. Selecting a Map token opens its child map.
  • Marker types: Classify markers as location, quest, danger, treasure, or note. The selected type appears as a badge; use the type filter to reduce clutter.
  • Measurements: Choose a distance template, grid-cell size, scale, and unit, then draw persistent measurements. Use Clear Measurements to remove them from the current map.
  • Pan & zoom: Middle-drag to pan; mouse wheel to zoom. View state persists per map.
  • Inspector: Click a token to view summary, notes, and quick stats; switch tabs to review more context.

  • Export Scenarios: Generate shareable scenario documents from selected records.
  • Campaign Dossier: Assemble a broader export package covering scenario and campaign context for prep or sharing.

  • Newsletters & handouts: Build in-world newsletter-style outputs and other player-facing handouts.

Dice Roller

Use the full Dice Roller for formula-based rolls with polyhedral previews, or the compact Dice Bar for always-on-top quick rolls.

  • Open: GM Tools → Dice for the full roller, or View → Show Dice Bar for the compact bar.
  • System presets: Supported dice and default formulas adapt to the selected campaign system.
  • Formula entry: Build expressions (e.g., 2d20+5), double-click presets to add dice, press Enter or click Roll.
  • Exploding: Toggle exploding dice to reroll max results.
  • Results & history: See grouped breakdowns and totals; recent rolls are logged for reuse.

Audio & Music

Books

Store and browse PDFs tied to your campaign. Imported books are indexed so you can search and reference excerpts.

  • Open: Campaign Workshop → Manage Books.
  • Import: Add individual PDFs or a whole folder. Indexing runs in the background.
  • View: Double-click a row or choose Open Book to launch the viewer. Use page navigation, zoom, and find (Prev/Next).
  • Excerpts: Export page ranges to assets/books/excerpts; excerpts appear in the Excerpts column for quick access.

Web Viewer

The project includes a lightweight web server for remote viewing and collaboration.

  • Start: Run python -m modules.web.GM_webviewer (or start_webserver.sh on Linux/macOS).
  • Login: Register an account, then browse NPCs, Places, Factions, News, Journals, and Clues.
  • Clues board: Drag clues, create links, and save positions.
  • Media: Portraits and uploaded assets are served from the active campaign.
Network safety: Do not expose the server directly to the public internet without authentication, access controls, and an appropriate reverse proxy. Player-facing services may expose campaign media.

Guided Tour

Choose Help → Guided Tour or Start Guided Tour in the Utilities sidebar. The tour highlights stable interface targets and guides a new GM through initial campaign setup. You can close it and restart it later from Help.

Troubleshooting & Safety

AI or Ollama models do not appear

Confirm Ollama is running, the configured base URL is reachable, and at least one model is installed. Reopen the generator to repeat background model discovery. Check the application logs if discovery still fails.

SwarmUI portrait generation fails

Re-select the SwarmUI directory from File → Set SwarmUI Path, verify SwarmUI starts independently, and try one entity before running bulk generation.

Media is missing

Keep campaign media inside the active campaign directory. If a campaign was moved manually, use the normal bundle/backup workflow or repair affected paths from the entity editor.

Imports and duplicate records

Back up first, review merge prompts, then run Campaign → Verify Campaign after the import.

Backup and restore

Create backups from File → Create Backup. Restoring replaces active campaign data and assets with the selected archive, so preserve the current campaign with a fresh backup first.

Player-facing displays

Review panels, maps, notes, portraits, and browser-accessible media before broadcasting. Use access tokens and trusted networks for web map and whiteboard services.

Tips

Documentation refresh: Run python scripts/generate_docs.py from the repository root after UI, menu, screenshot, or API changes. It refreshes docs/index.html, this manual, and docs/images/.
Exports: Use Export Scenarios, Campaign Dossier, Session Brief, or Export for Foundry depending on the audience.
Portrait workflow: Generate or link portraits from GM Tools, a scenario context menu, or an entity editor; double-click a portrait in a list to pop it out.
Cross-campaign reuse: Use the Cross-campaign Asset Library to move NPCs, objects, and maps between campaigns with their media.
AI settings: Use File → AI Settings for the endpoint, model, temperature, token limit, and optional API key.
Auto-improvement: Open File → Auto Improve App. Prefer dry-run until its Codex CLI and validation commands have been verified.
Logging: Enable logs in config/config.ini to troubleshoot AI imports and automated workflows.
